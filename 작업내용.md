## 수정 이력

### 1. run.bat — LF → CRLF 줄바꿈 수정
- **증상**: `'vel'은(는) 내부 또는 외부 명령...` 오류
- **원인**: cmd.exe는 CRLF 필요, 파일이 Unix LF로 저장됨
- **수정**: PowerShell로 CRLF 재저장

### 2. GitHub 업로드 실패 — 관련없는 히스토리 충돌
- **증상**: push 거부 (non-fast-forward)
- **원인**: 원격에 README.md만 있는 별도 히스토리 존재
- **수정**: `git pull origin main --allow-unrelated-histories -X ours`

### 3. GitHub Push Protection — .env 토큰 노출
- **증상**: push 시 시크릿 감지로 차단
- **원인**: .env 파일이 초기 커밋에 포함됨
- **수정**: `git filter-branch`로 히스토리에서 .env 제거 후 force push
- **조치**: .gitignore에 .env 추가, .env 로컬 재생성

### 4. 대시보드 데이터 로드 실패
- **증상**: "data.csv 로드 실패: [object ProgressEvent]"
- **원인**: 브라우저가 file:// 프로토콜에서 XHR 차단
- **수정**: index.html에 GitHub Raw URL 3번째 fallback 추가
```javascript
const GITHUB_RAW = "https://raw.githubusercontent.com/jeoniiiii123-art/88tiger/main/output/data.csv";
// 로드 순서: output/data.csv → data.csv → GITHUB_RAW
```

### 5. 주차 라벨 버그 — 5월 1주차가 10일
- **증상**: 5월 1주차에 5/1~5/10 전부 포함 (10일)
- **원인**: 월 첫날이 월요일이 아닌 경우 1주차 구간 계산 오류
- **수정**: `get_week_label()` 함수 로직 교정
  - 월 첫날이 월요일이면: 그 날부터 1주차
  - 월 첫날이 월요일이 아니면: 첫 월요일 이전 = 1주차, 첫 월요일부터 = 2주차

### 6. BA/NCA 날짜 오류 — 모든 행이 같은 날짜
- **증상**: BA/NCA 전체 행이 파일명의 시작일로 표시됨
- **원인**: `date_from_filename()` 사용 (파일명 날짜 고정)
- **수정**: `load_ba()`, `load_nca()`에서 각 행의 `날짜` 컬럼을 직접 파싱

### 7. BA 브랜드 전부 "BA광고"로 표시
- **증상**: BA 데이터 브랜드 컬럼이 전부 "BA광고"
- **원인**: 브랜드명 하드코딩
- **수정**: 상품명 → 광고명 → 캠페인명 순서로 브랜드 fallback 탐지

### 8. 엑셀 임시파일 PermissionError
- **증상**: `~$파일명.xlsx` 접근 시 오류로 파이프라인 중단
- **원인**: 엑셀이 열려있으면 `~$` 접두사 임시파일 생성
- **수정**: glob 시 `not f.name.startswith("~$")` 필터 추가

### 9. 6월 데이터 2배 집계 (중복 파일)
- **증상**: 6월 타이거모닝 PA 광고비 9,848,970 (실제 5,167,774의 약 2배)
- **원인**: `20260601_20260614`, `20260601_20260615` 두 파일이 동시 적재
- **수정**: `exclude_superseded()` 함수 추가
  - 같은 시작일이면서 끝날짜가 더 짧은 파일 자동 제외
  - 버그 수정: `os != s` → `(os, oe) != (s, e)` (시작일 동일한 경우 처리)
- **결과**: 9,028행 → 6,898행, 수치 정상화

### 10. 대시보드 완전 빈 화면 — Babel 버전 문제
- **증상**: 비밀번호 입력 후 흰 빈 화면
- **원인**: `@babel/standalone` 최신 버전이 ES 모듈 형식 출력 → 브라우저 거부
- **수정**: Babel 버전 고정 `@babel/standalone@7.23.10`

### 11. GitHub Pages 비활성화
- **증상**: "There isn't a GitHub Pages site here"
- **원인**: 저장소가 Private으로 변경 → 무료 플랜에서 Pages 자동 비활성화
- **수정**: API로 저장소 Public 전환 후 Pages 재활성화

### 12. coupang_download.py — 월초 실행 시 날짜 역순 버그
- **증상**: 7월 1일 실행 시 start=7/1, end=6/30 → 역순으로 다운로드 실패
- **원인**: `start = today.replace(day=1)` (오늘 기준 달 1일)
- **수정**: `start = end.replace(day=1)` (어제 기준 달 1일)

### 13. coupang_download.py — 달력 이전 달 이동 버그
- **증상**: 달력이 2002년으로 이동 (24년 이전)
- **원인**: loop 12회 × 2(시작/종료일) = 24회 버튼 클릭, 이전 년도(<<) 버튼을 잘못 클릭
- **수정**: 
  - `_click_prev_month_btn()` 메서드 신설
  - `super` 클래스 제외한 정확한 XPath로 이전 달(< ) 버튼만 클릭
  - 오늘과 목표 달의 차이를 계산해 **정확한 횟수**만 클릭

## 주요 파일 설명

### coupang_pipeline.py
- `get_week_label(d)`: 날짜 → "N월 M주차" 변환 (월요일 시작)
- `get_brand(text)`: 텍스트에서 브랜드명 추출 (타이거모닝/팔팔호랑이/기타)
- `load_pa(path)`: PA 엑셀 로드 (옵션ID 단위 집계)
- `load_nca(path)`: NCA 엑셀 로드 (행별 날짜 파싱)
- `load_ba(path)`: BA 엑셀 로드 (행별 날짜 파싱)
- `exclude_superseded(files)`: 날짜 범위가 더 긴 파일이 있으면 짧은 파일 제외
- `run_pipeline()`: 전체 실행 진입점

### coupang_download.py
- Selenium으로 쿠팡 광고 플랫폼 자동 로그인
- PA/NCA/BA 보고서 각각 생성 요청 → 다운로드 → RAW 폴더 저장
- 기본 기간: 이번 달(어제 기준) 1일 ~ 어제
- `_click_prev_month_btn()`: 달력에서 이전 달 버튼(< 만, << 제외) 클릭
- `_set_dates()`: 목표 월까지 정확한 횟수만 이동 후 날짜 선택

### index.html
- React + Babel (7.23.10 고정) 싱글 페이지 앱
- 필터: 월 / 브랜드 / 광고유형 / 주차별
- 데이터 로드 순서: `output/data.csv` → `data.csv` → GitHub Raw URL

1. `run.bat` 더블클릭
2. 크롬 브라우저 자동 실행 → 쿠팡 광고 플랫폼 로그인 → 보고서 생성/다운로드
3. data.csv 자동 생성
4. GitHub 자동 push
5. 대시보드 1~2분 후 반영
